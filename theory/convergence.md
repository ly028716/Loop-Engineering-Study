# Convergence

收敛表示状态到目标的误差持续减小，而非仅仅“运行了很多轮”。数值目标中，
`GoalEvaluator` 使用绝对误差计算分数 `1 / (1 + error)`；当误差为零时，
评分为 1 且评估成功。限幅增量策略避免超过目标，因此可以稳定地逼近。

[basic_loop](../experiments/basic_loop.py) 的值按 `0 → 2 → 4 → 5` 变化，得分
序列为 `0.25 → 0.5 → 1.0`。这是可检查的单调收敛例子。

收敛不等于评估一定可信。[repair_loop](../experiments/repair_loop.py) 的动作让
状态达到目标，但故障评估器报告 0 分和未成功。它提醒我们应同时检查真实状态、
动作结果和评估结果，而不要只相信单一指标。
