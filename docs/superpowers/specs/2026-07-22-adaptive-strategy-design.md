# 自适应策略与预算分配实验设计

## 目标

本实验研究自适应策略控制器（adaptive strategy controller）能否在固定的安全步骤预算内，
比现有策略使用更少的步骤达到任务目标。

实验沿用项目现有的循环、Trace 和 Artifact 机制，不引入外部依赖，也不修改核心运行框架。

## 对比对象

实验在同一确定性任务上运行以下四种策略：

| 策略 | 行为 |
| --- | --- |
| `fixed` | 始终使用固定步长推进。 |
| `error_aware` | 根据当前误差选择推进步长。 |
| `memory_aware` | 根据近期 Trace 中的进展与失败信息选择步长。 |
| `adaptive` | 根据误差、连续失败、进展趋势和剩余预算，在保守、快速与恢复推进间切换。 |

## 实验场景

任务从较大的初始误差开始。接近目标时，大步长会导致低效或失败；在中途会注入一次可恢复的动作失败。

该场景同时要求策略具备三种能力：

1. 初始误差大时快速缩小误差；
2. 接近目标时降低步长，避免无效推进；
3. 出现失败时切换到恢复推进，并在预算内回到正常路径。

任务是完全确定性的，确保不同策略的结果可复现。

## 自适应控制器

每轮评估完成后，控制器从最新状态和 Trace 中读取以下信号：

- 当前误差；
- 是否发生动作失败；
- 连续未取得进展的轮数；
- 剩余安全步骤预算；
- 最近一次动作结果。

控制器选择一个明确的策略模式：

| 模式 | 触发条件 | 动作意图 |
| --- | --- | --- |
| `fast` | 误差较大且运行稳定 | 使用较大步长，减少达到目标前的轮数。 |
| `careful` | 接近目标 | 使用较小步长，避免越界或低效修正。 |
| `recovery` | 本轮失败或连续无进展 | 使用可恢复的推进方式，优先重新获得进展。 |
| `budget_guard` | 剩余预算较少 | 限制动作幅度，避免无意义消耗步骤。 |

策略选择结果必须写入动作元数据或 Trace 可读取字段，以便统计切换次数并支持复盘。

## 输出与 Artifact

`experiments/adaptive_strategy.py` 提供 `run_comparison()`，默认将结果写入
`.loop/runs/adaptive-strategy/`。每种策略产生一个可通过
`load_run_artifact()` 加载的 Artifact。

每条比较结果至少包含：

- `strategy`：策略名称；
- `status` 与 `success`：运行状态和是否达到目标；
- `steps`：执行步骤数；
- `final_score`：最终评估分数；
- `switch_count`：自适应策略的模式切换次数；
- `recovery_count`：恢复模式使用次数；
- `stop_reason`：停止原因；
- `artifact_path`：Artifact 路径。

非自适应策略的 `switch_count` 和 `recovery_count` 为 `0`。

## 验收条件

1. 四种策略都能在安全预算内完成确定性任务；
2. `adaptive` 策略成功且步骤数严格少于 `fixed`；
3. `adaptive` 至少发生一次策略模式选择，并在注入失败后进入恢复模式；
4. 所有 Artifact 均可加载，且保留 Trace；
5. 新增单元测试、实验说明、实验导航、双语 README 和进度记录；
6. 完整 `pytest -q` 测试套件通过。

## 非目标

- 不实现机器学习或在线训练；
- 不增加并发、多智能体协调或外部服务；
- 不改变已有实验和核心循环的公共接口。
