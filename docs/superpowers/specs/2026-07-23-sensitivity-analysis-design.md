# Loop Engineering 参数敏感性分析设计

## 目标

在现有策略评测基础上，采用单参数扫描定位策略排行发生变化的边界条件。重点是解释排名何时翻转，而不是寻找单一最佳参数组合。

实验保持确定性，不引入第三方依赖，也不修改 `loop_engineering/` 公共接口。

## 参数族与基线

每次只改变一个参数，其余参数固定为基线：目标距离 `6`、失败发生在第 `2` 次动作尝试、最大步骤预算 `8`。

| 参数族 | 档位 | 基线值 |
| --- | --- | --- |
| `goal_distance` | `4`、`6`、`8` | `6` |
| `failure_attempt` | `1`、`2`、`3` | `2` |
| `step_budget` | `3`、`5`、`8` | `8` |

因此分析固定包含 9 个配置；每个配置运行 `fixed`、`error_aware`、`memory_aware` 和 `adaptive` 四种策略，共 36 次运行。

## 运行与结果

新增 `experiments/sensitivity_analysis.py`，提供：

- `run_sensitivity(output_dir: str | Path = ".loop/runs/sensitivity-analysis")`；
- 每个配置的参数族、参数值、策略明细和稳定排行榜；
- 每条策略运行的 Trace 与独立 Artifact；
- 每个参数族的相邻档位比较和首次排名翻转点。

运行配置至少包含：`parameter_family`、`parameter_value`、`baseline`、`ranking`、`runs` 和 `artifact_paths`。

每条策略结果至少包含：`strategy`、`success`、`steps`、`final_score`、
`failure_count`、`recovery_count`、`budget_respected`、`total_score`、`rank` 和 `artifact_path`。

## 评分与排序

每个配置使用当前基准相同的 100 分评分结构：成功率 50 分、步骤效率 25 分、恢复能力 15 分、预算遵守度 10 分。

本阶段每个配置只有一个确定性任务，因此：

- 成功率为 0 或 1；
- 步骤效率以该配置中成功策略的最少步骤为基准；
- 恢复能力要求运行成功且出现至少 1 次恢复决策；
- 预算遵守度为 0 或 1。

排行榜按总分降序、步骤数升序、策略名升序排列。`rank` 从 1 开始。

## 翻转定义

对于同一参数族的相邻档位，比较所有策略对的相对顺序。例如前一档中 `adaptive` 位于 `error_aware` 之前，而后一档相反，则该策略对发生翻转。

首次翻转点是某策略对首次与前一档相对顺序不同的档位值。若某参数族所有相邻档位的相对顺序均不变，则该族返回空翻转列表和 `no_flip=True`。

## 验收条件

1. 固定生成 9 个单参数配置、36 次运行和 36 个可加载 Artifact；
2. 每个配置只改变声明的参数，其他两个参数等于基线；
3. 每个配置输出稳定排行榜和从 1 开始的排名；
4. 翻转分析仅比较同一参数族中的相邻档位，并可重复运行；
5. 对无翻转参数族显式报告 `no_flip=True`；
6. 新增单元测试、学习文档、实验导航、双语 README 和进度记录；
7. 完整 `pytest -q` 测试套件通过。

## 非目标

- 不执行多参数笛卡尔积或随机采样；
- 不将单参数结果解释为真实生产环境的因果结论；
- 不修改现有基准或既有实验的公共行为。
