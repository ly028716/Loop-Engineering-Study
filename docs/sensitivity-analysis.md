# 参数敏感性分析

参数敏感性分析在既有策略评测基础上执行单参数扫描，用来定位四种策略的相对排名何时发生变化。它描述的是本项目确定性实验中的边界条件，而不是对真实生产环境作因果或泛化结论。

## 基线与参数族

基线配置为目标距离 `6`、动作失败发生在第 `2` 次尝试、最大步数预算 `8`。每次实验只改变一个参数，其他两个参数保持基线值：

| 参数族 | 扫描值 | 基线值 |
| --- | --- | ---: |
| `goal_distance` | `4`、`6`、`8` | `6` |
| `failure_attempt` | `1`、`2`、`3` | `2` |
| `step_budget` | `3`、`5`、`8` | `8` |

因此会生成 9 个配置；每个配置运行 `fixed`、`error_aware`、`memory_aware` 与 `adaptive` 四种策略，共输出 36 条运行记录和 36 份可回放 Artifact。

## 运行实验

```powershell
python experiments/sensitivity_analysis.py
```

脚本输出 JSON，其中每个配置都带有参数族、参数值、基线、局部运行记录、Artifact 路径和稳定排名。Artifact 默认写入 `.loop/runs/sensitivity-analysis/`，可使用 `loop_engineering.artifacts.load_run_artifact()` 加载 Trace 与指标。

每个配置采用单一确定性任务，并按 100 分结构评分：成功 `50` 分、相对最快成功步数的效率 `25` 分、成功且至少一次恢复决策的恢复能力 `15` 分、预算内停止 `10` 分。排名按总分降序、步数升序、策略名升序生成，名次从 `1` 开始。

## 阅读排名翻转

分析只比较同一参数族的相邻档位，例如 `goal_distance` 的 `4 → 6` 与 `6 → 8`。对于每一对策略，若其前后相对顺序发生改变，就记录 `from_value`、`to_value` 和策略对。

`flip_analysis` 中每个参数族都有 `flips` 与 `no_flip`：空 `flips` 同时意味着 `no_flip=true`。这使学习者能够区分“在当前扫描范围内排名稳定”与“在某个相邻边界发生翻转”。

## 解释边界

这些结果仅来自确定性动作、固定策略与有限离散档位。它们适合用于检查 Loop Engineering 中目标难度、故障时机和预算约束如何影响策略排序；不应用于推断多参数联合作用、随机任务分布或真实服务可靠性。
